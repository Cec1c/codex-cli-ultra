# Codex CLI i18n MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. The user selected inline execution. Steps use checkbox syntax for tracking.

**Goal:** Build a reproducible Codex CLI 0.144.1 i18n proof of concept that extracts real TUI messages, compiles a zh-CN Fluent pack, applies a version-locked Rust bridge to an isolated Codex source worktree, renders several translated status-line setup strings, and falls back to compiled English on every catalog failure.

**Architecture:** Language authors maintain FTL. The JavaScript executor validates FTL and compiles the currently supported plain messages into a JSON runtime catalog. A thin Rust module in codex-tui reads that catalog once and returns the existing English string when the file, JSON, key, or translated value is invalid. A version adapter applies exact, preflighted source transformations only to commit 44918ea10c0f99151c6710411b4322c2f5c96bea.

**Tech Stack:** Node.js 24, npm, @fluent/bundle 0.19.1, Node built-in test runner, PowerShell 7, Rust 1.96.1, serde_json already present in codex-tui, Git worktrees, insta snapshots.

## Global Constraints

- Stable target: Codex CLI 0.144.1, upstream tag rust-v0.144.1, commit 44918ea10c0f99151c6710411b4322c2f5c96bea.
- MVP-wired UI: the four strings in codex-rs/tui/src/bottom_pane/status_line_setup.rs for theme colors, active theme description, setup title, and setup description.
- Preliminary catalog: the four wired status-line messages plus six onboarding/auth messages already verified in 0.144.1.
- Language source: declarative Fluent/FTL. Language packs execute no JavaScript.
- Runtime artifact: generated JSON containing only validated plain messages. This is an MVP bridge artifact, not the long-term language-authoring format.
- Fallback: missing environment variable, missing file, invalid JSON, missing key, or empty value returns the existing English string without panic.
- Unsupported or drifted Codex source is never modified.
- Development and Codex source testing occur in isolated worktrees.
- Windows 11 and PowerShell 7 are the first operator path.
- The Worked for duration easter egg is documented as the next feature and is not wired into this MVP.
- Repository commits use Chinese Conventional Commit messages in the form feat: 中文说明.

- [ ] **Step 3: Implement the minimal extractor**

message-specs.mjs defines ten records with id, ftlKey, surface, kind, translation, mvpStatus, repository-relative path, Rust symbol, exact quoted source anchor, and English text.

extract.mjs must:

~~~javascript
export async function extractCatalog(sourceRoot, specs) {
  const records = [];
  for (const spec of specs) {
    const source = await readFile(join(sourceRoot, spec.path), "utf8");
    const indexes = findAllIndexes(source, spec.anchor);
    if (indexes.length !== 1) {
      throw new Error(
        `${spec.id}: expected exactly one source anchor, found ${indexes.length}`
      );
    }
    const line = source.slice(0, indexes[0]).split(/\r?\n/).length;
    const fingerprintPayload = `${spec.path}|${spec.symbol}|${spec.english}`;
    records.push({
      schemaVersion: 1,
      id: spec.id,
      ftlKey: spec.ftlKey,
      surface: spec.surface,
      english: spec.english,
      kind: spec.kind,
      placeholders: spec.placeholders ?? [],
      richSlots: spec.richSlots ?? [],
      translation: spec.translation,
      mvpStatus: spec.mvpStatus,
      source: {
        repository: "openai/codex",
        release: "rust-v0.144.1",
        path: spec.path,
        symbol: spec.symbol,
        line,
        fingerprint: "sha256:" + createHash("sha256")
          .update(fingerprintPayload)
          .digest("hex")
      },
      firstSeen: "0.144.1",
      lastVerified: "0.144.1"
    });
  }
  return records.sort((a, b) => a.id.localeCompare(b.id));
}
~~~

writeCatalogArtifacts writes one compact JSON object per JSONL line and generates a Chinese-first Markdown table grouped by mvpStatus.
