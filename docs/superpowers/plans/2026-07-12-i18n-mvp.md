# Codex CLI Ultra i18n MVP 计划索引

> 本索引依据 `docs/superpowers/specs/2026-07-12-i18n-build-distribution-design.md`，取代旧的单体实施计划。必须按顺序执行三个子计划。

## 执行顺序

1. [Rust i18n 运行时 MVP](2026-07-12-i18n-runtime-mvp.md)

   建立源码文本目录、外部 FTL 语言包、版本锁定适配器、Rust Localizer、隐藏 JSON 自检、三档 CJK 快照、状态栏翻译以及 `加班了 {duration}` 参数化消息。产出一个在隔离上游工作树中通过测试并记录同 profile 体积增量的补丁构建。

2. [并排启动器与安装器 MVP](2026-07-12-i18n-launcher-installer-mvp.md)

   建立本地状态、官方安装发现、启动目标选择、JavaScript 启动器、安全下载与解压、事务式安装、PATH 管理、locale、doctor、升级、卸载和故障注入测试。

3. [预编译 Release 与端到端 MVP](2026-07-12-i18n-release-e2e-mvp.md)

   使用官方 Codex package builder 构建完整 Windows x64 包，生成 Release 清单、SHA-256、许可证和对应源码归档，通过 GitHub Actions 产出预发布资产，并完成真实安装、回退和卸载验证。

## 共同验收边界

- 固定上游目标：Codex CLI `0.144.1`、标签 `rust-v0.144.1`、提交 `44918ea10c0f99151c6710411b4322c2f5c96bea`。
- 正常 `codex` 启动不联网。
- 外部语言包只包含声明式数据，不执行代码。
- 单条翻译失败回退该条英文，整包失败回退完整英文。
- Ultra 不兼容、缺失或损坏时运行官方绝对二进制。
- 不物理覆盖或删除官方 npm 安装。
- Release 保留上游 canonical package 的 `bin`、`codex-resources` 和 `codex-path` 完整布局。
- 合并和发布必须通过 Node、Rust、对应源码、真实安装、英文回退、官方回退和卸载 CI 门禁。
- 所有项目提交使用中文 Conventional Commit，例如 `feat: 添加 Rust i18n 运行时`。

## 计划之间的交付契约

计划 1 产出：

- `research/codex-0.144.1/tui-messages.jsonl`
- `packages/languages/zh-CN/`
- `src/adapter/` 与 `adapters/codex/0.144.1/`
- `codex --ultra-i18n-self-check` 的中文与英文回退 JSON 证据
- `research/codex-0.144.1/i18n-size.json`
- 可重复应用并通过 Rust 测试的上游补丁

计划 2 消费计划 1 的语言包与兼容元数据，产出：

- `src/launcher/`、`src/installer/`、`src/commands/`
- 无网络依赖的 launcher bundle 与独立管理 executor bundle
- `scripts/install.ps1` 与 `scripts/uninstall.ps1`
- 可对本地 fixture Release 完成安装、启动、回退和卸载的 JavaScript 执行器

计划 3 消费前两个计划的适配器、语言包和安装器，产出：

- Windows x64 Codex Ultra package archive
- Release manifest、SHA-256、许可证集合和对应源码归档
- GitHub Actions 构建与预发布工作流
- 从 Release 资产完成安装、中文运行证据、官方回退和卸载的端到端记录
